# 🔒Security Policy

## 🔄Supported Versions

We actively address security issues for the latest version of this project.  
If you are using an older version, we recommend upgrading to the latest release to benefit from security fixes.

---

## 🐞Reporting a Vulnerability

We take security seriously and appreciate your efforts in responsibly disclosing vulnerabilities. If you find a security issue, please follow these guidelines:

1. **Do Not Disclose Publicly**:  
   Do not create issues or pull requests on GitHub for security-related matters. Publicly disclosing vulnerabilities before they are addressed can put users at risk.

2. **Contact Us Directly**:  
   To report a vulnerability, please contact any of the following maintainers:

- [DahlByk] - [<dahlbyk@gmail.com>](mailto:dahlbyk@gmail.com)
- [Jrusbatch] - [<jrusbatch@gmail.com>](mailto:jrusbatch@gmail.com)
- [Ritwik] - [<ritwik.1204@gmail.com>](mailto:ritwik.1204@gmail.com)

Please include detailed information about the vulnerability, including steps to reproduce and potential impact.
   - A description of the issue.
   - Steps to reproduce the vulnerability.
   - Any potential impact or risk assessment.

3. **Acknowledgment**:  
   We will acknowledge receipt of your report within 48 hours and provide feedback on the next steps.

4. **Resolution Timeline**:  
   Once we confirm the issue, we aim to provide a resolution as fast as we can. We will notify you once a fix has been released.
